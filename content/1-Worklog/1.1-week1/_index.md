### Week 1 Objectives:

- Get acquainted with the **First Cloud AI Journey** program, participating members, and the learning path throughout the practice.
- Understand the overview of **AWS**, **Cloud Computing**, and the role of cloud computing in deploying, operating, and scaling IT systems.
- Master the basic AWS service groups such as **Compute**, **Storage**, **Networking**, **Database**, **Security**, and **Management Tools**.
- Get familiar with **AWS Free Tier**, **AWS Management Console**, and basic operations when using an AWS account.
- Learn about **AWS CLI** and how to use the command line to interact with AWS resources at a basic level.
- Learn how to represent a general AWS architecture diagram, including the main components in a cloud system.

### Week 1 Achievements:

**Overall:**

In the first week, I got acquainted with the **First Cloud AI Journey** program, understood the learning objectives and the contents to be completed in the initial phase. I grasped the overall picture of AWS, the role of Cloud Computing in enterprises, and how AWS services are organized by functional groups.

Additionally, I got familiar with the AWS Free Tier account, AWS Management Console interface, AWS CLI, and how to describe basic AWS architecture diagrams. This is an important foundation to continue with more in-depth labs in the following weeks.

**Learned Theory:**

- Understand the concepts of **AWS** and **Cloud Computing**.
- Grasp the role of cloud computing in:
  - Flexible system scaling.
  - Rapid resource deployment.
  - Reducing initial infrastructure investment costs.
  - Centralized resource management.
  - Supporting multiple different service models.
- Grasp the basic AWS service groups:
  - **Compute**: process and run applications.
  - **Storage**: store data and resources.
  - **Networking**: connect and manage networks.
  - **Database**: manage databases.
  - **Security**: secure accounts, access permissions, and resources.
  - **Management Tools**: monitor, manage, and operate systems.
- Understand the significance of **AWS Free Tier** during learning and practice.
- Grasp the role of **AWS Management Console** in managing AWS resources through a web interface.
- Understand the role of **AWS CLI** in interacting with AWS resources using the command line.
- Grasp the components commonly appearing in AWS architecture diagrams:
  - User/Client.
  - Region.
  - Availability Zone.
  - VPC.
  - Public Subnet / Private Subnet.
  - Internet Gateway.
  - NAT Gateway.
  - EC2.
  - Load Balancer.
  - Auto Scaling Group.
  - RDS.
  - S3.
  - IAM.
  - CloudWatch.

**Practiced with Labs:**

- Got acquainted with the program and learning path of First Cloud AI Journey.
- Learned about and prepared an AWS Free Tier account for the practice process.
- Accessed the AWS Management Console and got familiar with the resource management interface.
- Practiced searching, accessing, and observing AWS services on the Console.
- Installed and configured AWS CLI on a personal computer.
- Configured the necessary information for AWS CLI:
  - Access Key.
  - Secret Key.
  - Default Region.
  - Output format.
- Practiced some basic AWS CLI commands:
  - Check configuration information.
  - View region list.
  - Check EC2 service information.
  - Create and manage key pairs.
  - Observe active resources.
- Practiced sketching a general AWS architecture diagram, showing the main layers:
  - Network.
  - Compute.
  - Database.
  - Storage.
  - Security.
  - Monitoring.
- Capable of describing a basic AWS architecture at a high level, including the access flow from users to the system and the role of each main service group.
