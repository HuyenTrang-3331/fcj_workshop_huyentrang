### Week 2 Achievements:

**Overall:**

In Week 2, I completed the learning content of **Module 1** in the **First Cloud AI Journey** program. This week's content helped me better understand the AWS platform, the differences between AWS and traditional infrastructure models, and how to start learning and practicing with AWS services.

In addition to the theory, I practiced initial AWS account setup steps like creating an account, enabling MFA, creating an Admin Group and Admin User, and learning about account authentication support. I also practiced creating various types of AWS Budgets to control costs, while learning about AWS Support plans and how to manage support requests.

**Learned Theory:**

* Understand **Cloud Computing** concepts and the role of cloud computing in enterprises.
* Master key differences between AWS and traditional infrastructure models:

  * Flexible scalability.
  * Rapid resource deployment.
  * Pay-as-you-go cost model.
  * High availability.
  * Diverse service ecosystem.
  * Centralized operations and management capabilities.
* Understand how to start the AWS learning journey, including:

  * Identifying required foundational knowledge.
  * Getting familiar with an AWS account.
  * Learning about core services.
  * Combining theory, labs, and technical note-taking.
* Master AWS global infrastructure:

  * **Region**: geographical area where AWS services are deployed.
  * **Availability Zone**: availability zone that helps increase system stability and availability.
  * **Edge Location**: edge location that supports content delivery and optimizes user experience.
* Understand the role of global infrastructure in building scalable, stable, and more fault-tolerant systems.
* Master AWS service management tools:

  * **AWS Management Console**: manage resources via a web interface.
  * **AWS CLI**: interact with AWS resources using the command line.
  * **AWS SDK**: integrate and manage AWS Services through programming languages.
* Understand the role of **AWS Budgets** in:

  * Tracking costs.
  * Controlling service usage levels.
  * Alerting when costs exceed thresholds.
  * Limiting unexpected costs.
* Master basic budget types:

  * Cost Budget.
  * Usage Budget.
  * Reservation Instance Budget.
  * Savings Plans Budget.
* Understand the role of **AWS Support** and differences between support plans.
* Know about types of support requests and how to use AWS Support during operations or incident handling.
