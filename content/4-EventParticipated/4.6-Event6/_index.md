# "AWS Enterprise Cloud Architectures and Industry Application" Event Summary

## Event Objectives

The **AWS Enterprise Cloud Architectures and Industry Application** event was organized to share practical knowledge about enterprise cloud architectures and how AWS is applied across various industries.

Through the speakers' presentations, participants had the opportunity to better understand how enterprises build cloud systems, leverage data, undergo digital transformation, optimize operations, and ensure security on the AWS platform.

## General Information

- **Time:** 09:00, July 04, 2026
- **Location:** 26th Floor, Bitexco Building, 02 Hai Trieu Street, Saigon Ward, Ho Chi Minh City
- **Event Name:** Enterprise Cloud Architectures and Industry Application
- **Role:** Participant

## List of Speakers

- **Nguyen Gia Hung** – Shared an overview of enterprise cloud architecture and AWS's role in digital transformation.
- **Banh Cam Vinh** – Shared about data, cloud systems, and technical applications in enterprise environments.
- **Nhu Tran** – Shared about AWS's role in supporting enterprises to build suitable technology solutions.
- **Khang Nguyen** – Shared about practical cloud architecture and how to apply AWS in enterprise systems.

## Key Highlights

### Overview of Enterprise Cloud Architecture

The opening session by Mr. Nguyen Gia Hung helped participants get an overview of cloud architecture in enterprises. Cloud is not just a place to store data or run servers, but also a platform to help enterprises innovate, scale systems, and accelerate digital transformation.

An effective cloud architecture needs to ensure elements like availability, scalability, security, performance, monitoring, and cost optimization. This content helps understand that when designing systems on AWS, a holistic view is needed rather than just learning individual services.

### Data and Cloud Systems in Enterprises

The sharing session by Mr. Banh Cam Vinh emphasized the important role of data in modern enterprises. Data can be collected from websites, applications, transaction systems, system logs, or internal platforms.

In cloud environments, data needs to be stored, processed, analyzed, and protected properly to create value for the enterprise. This content helps understand more about the relationship between cloud and data, especially in areas like Data Engineering, Data Analytics, and reporting systems.

### AWS's Role in Digital Transformation

The sharing session by Ms. Nhu Tran focused on AWS's role in supporting enterprises' digital transformation. AWS provides many services for computing, storage, databases, networking, security, and system monitoring.

Using AWS helps enterprises reduce the burden of managing physical infrastructure, deploy products faster, and scale resources flexibly according to actual needs. Through this, it is understood that cloud not only supports technically but also contributes to creating business value.

### Practical Cloud Architecture on AWS

The sharing session by Mr. Khang Nguyen helped participants better understand how to design cloud architecture for enterprises. A practical cloud system needs to combine many components like VPC, EC2, S3, RDS, IAM, CloudWatch, Load Balancer, and Auto Scaling.

This content shows that moving systems to the cloud is not just about deploying applications, but also needs to consider security, operations, scalability, and cost. A good architecture must both meet technical requirements and align with the enterprise's business goals.

### Security, Operations, and Cost Optimization

The event also emphasized the importance of security and operations on the cloud. When using AWS, enterprises need to manage access rights with IAM, log, monitor systems, encrypt data, and detect security risks.

In addition, cost optimization is also an important factor. Cloud is not automatically cheap if used incorrectly. Deployers need to monitor resources, turn off unused services, choose appropriate configurations, and control costs regularly.

## What You Learned

Through the event, several important lessons were learned:

- **Cloud needs to be designed strategically:** A good AWS system must ensure security, performance, scalability, and reasonable cost.
- **Data is an important asset:** Cloud helps enterprises store, process, and analyze data more effectively.
- **AWS supports digital transformation:** Enterprises can deploy products faster and more flexibly when using cloud.
- **Security must be integrated from the beginning:** IAM, logging, monitoring, and data encryption are necessary elements when building systems.
- **Cost optimization is continuous:** Need to regularly monitor resources and billing to avoid waste.

## Applying to Work

Knowledge from the event can be applied to studying, practicing AWS, and building personal projects.

When building a website or API, cloud thinking can be applied by dividing the system into clear parts like frontend, backend, database, storage, security, and monitoring. This helps the system be easier to maintain and scale.

AWS services like EC2, S3, RDS, Lambda, IAM, CloudWatch, and VPC can also be used to deploy practical projects. For example, an e-commerce website can use S3 to store images, RDS to store order data, IAM to assign permissions, and CloudWatch to monitor logs.

Additionally, data content helps have more ideas for building reporting dashboards, data analysis systems, or simple data processing workflows on the cloud.
