### Key Highlights

#### Docker – A Containerization Technology

This session introduced Docker and containerization technology. The speaker explained the difference between traditional virtual machines and containers, showing why Docker is widely used in modern software development.

Docker helps package applications with their dependencies and required environment so they can run consistently across different systems. Key concepts such as Docker Image, Docker Container, Dockerfile, image layers, build cache, and basic Docker commands were introduced.

The session also discussed Docker use cases in CI/CD pipelines, microservices, development/testing environments, cloud-native applications, and legacy system modernization.

#### Multiplayer in the Cloud: Connecting Godot Clients with AWS WebSockets

This session focused on building a cloud-based multiplayer game system using Godot and AWS WebSocket. The speaker introduced networking approaches such as UDP/ENet, WebSocket, and HTTP Polling.

WebSocket was presented as a suitable solution for turn-based games, lobby systems, chat features, and lightweight real-time mini games. The architecture included Godot Client, API Gateway WebSocket, AWS Lambda, DynamoDB, and CloudWatch.

DynamoDB was used to store connection information and player states, while Lambda handled connection, disconnection, matchmaking, game choices, and results. The demo showed two clients connecting, matching, sending Rock/Paper/Scissors choices, and receiving results in real time.

#### Combining AWS WAF with Machine Learning for Cyber Attack Detection on AWS

This session presented how AWS WAF can be combined with Machine Learning to detect cyber attack behaviors. AWS WAF protects web applications from common attacks using rule-based and signature-based mechanisms.

However, fixed rules may not detect new or complex attack patterns. Therefore, Machine Learning can be used to support anomaly detection and improve security monitoring. The speaker introduced a Network Intrusion Detection System model using the CSE-CIC-IDS2018 dataset.

The session also mentioned tools and services such as Python, scikit-learn, pandas, AWS WAF, Amazon S3, AWS Lambda, Kinesis Data Firehose, GuardDuty, Security Hub, and CloudWatch. Important topics included real-time dashboards, alert visualization, class imbalance, false positives, and model optimization.

#### AWS Neptune for Building a Graph Knowledge Base for GraphRAG

This session introduced GraphRAG and the use of AWS Neptune to build a graph knowledge base. The speaker first explained Retrieval-Augmented Generation, which helps large language models retrieve external knowledge before generating answers.

Traditional RAG is useful but may be limited when questions require understanding relationships between multiple entities. GraphRAG improves this by using graphs to store entities and relationships, supporting better multi-step reasoning.

The speaker presented two implementation approaches: using Amazon Bedrock Knowledge Bases with Neptune Analytics, or building a custom pipeline with tools such as LlamaIndex and storing the graph in Amazon Neptune.

#### From IT Helpdesk to Senior Sysadmin: First Steps Toward Cloud and DevOps

This session shared a practical career journey from IT Helpdesk to Senior Sysadmin and then toward Cloud/DevOps. The speaker explained that Helpdesk is an important foundation for learning troubleshooting, user communication, incident handling, and IT system operations.

As professionals move toward System Administrator roles, they need stronger skills in server management, networking, monitoring, security patching, backup, and automation.

For Cloud/DevOps, the speaker emphasized cloud mindset, AWS, elastic scaling, pay-as-you-go models, managed services, Infrastructure as Code with Terraform, version control, CI/CD, Docker, monitoring, and DevOps culture. The session also highlighted the importance of real projects, portfolios, and hands-on practice.

### Key Takeaways

Through the event, I gained several important lessons:

* **Docker and deployment consistency:** Docker helps applications run consistently across different environments and supports modern deployment workflows.
* **Real-time cloud architecture:** AWS WebSocket, Lambda, and DynamoDB can be used to build simple multiplayer or real-time applications.
* **Layered security mindset:** AWS WAF can block known threats, while Machine Learning can help detect abnormal behaviors.
* **GraphRAG value:** GraphRAG is useful for systems that need to understand relationships between entities, not only retrieve similar text.
* **Cloud/DevOps career path:** A strong foundation in troubleshooting, networking, servers, Docker, and automation is important before moving deeper into Cloud and DevOps.
